Zhivago.jl

Quantum architecture simulation using tensor networks

Zhivago.jl is a Julia package for simulating realistic, structured quantum architectures using tensor network methods (via ITensor). It provides flexible, architecture-aware circuit compilation, time evolution, and measurement workflows with support for noise, dissipation, and pulse-level control.

Key Capabilities

Predefined quantum architectures Build and explore complex device layouts (e.g. Planckian Ladder) using modular, hierarchical definitions derived from parent schemes.

Tensor-network time evolution Multiple engines for efficient many-body simulation:

  • TDVP
  • TEBD
  • DMRG-based methods

Quantum operations Apply single- and two-qubit gates, with explicit pulse-level control and customizable pulse envelopes.

Noise modeling Incorporate static inhomogeneity and other noise sources, and average over noise realizations to study robustness.

Open-system dynamics Simulate dissipative evolution using quantum trajectory / Monte Carlo wavefunction methods.

State preparation and measurement High-level routines for initialization, readout, and observable extraction.

External tool integration High-level APIs to connect to external tools, e.g. Livingston, Frankenstein, Bovary, and Metamorphosis modules from the ArchBuilder(–Link–) software stack.

Quick Start Example

using Zhivago

# 1. Set up a 2-logical-qubit ladder system (15 physical qubits total)
sys = setup_ladder_system(2)

# 2. Create and customize simulation parameters
params = configure_system_parameters()

# Adjust parameters as needed
params["use_trajectories"] = false     # closed-system evolution
params["zeta_nom"] = 200.0             # custom device / coupling parameter

# 3. Plan a logical Hadamard gate on logical qubit 1
ops, icc_col = plan_quantum_operation(sys, params;
    operation_type = "hadamard",
    logical_targets = [1],
    icc_col = 3,
)

# 4. Run the simulation with (optional) noise level control
result = run_simulation(sys, params, ops;
    noise_level = 0.0,
    icc_col = 3,
)

This example shows the full workflow:

  1. define an architecture,
  2. configure physics / control parameters,
  3. compile a high-level logical operation into low-level pulses / tensor ops,
  4. evolve and collect results.

Notes

  • Zhivago.jl is built on ITensor, so simulations scale using tensor network factorizations instead of brute-force Hilbert space methods.
  • Architectures and operations are expressed at the logical and physical level, enabling architecture-aware compilation.
  • Pulse-level descriptions and dissipation modeling make it suitable for studying controllability and fidelity in near-term / NISQ-era devices.

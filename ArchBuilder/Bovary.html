Bovary.bovary module¶

Bovary — Pulse Composer & Circuit Editor (Embedded Frame)¶

This module provides the Bovary embedded GUI used inside ArchBuilder. It offers: - Architecture loading (.arch + associated resources) - Gate palette (Logical / Tape / Analog / Error) - Circuit preview and QASM export - Optional QPU compiler integration (composites, validation) - Optional noise model editing (logical QEC) and handling

The main entry point is SubGUI, a ttk.Frame intended to be embedded by the parent application (ArchBuilder). It does not create a top-level window by itself.

class Bovary.bovary.ArchitectureSelectorFrame(parent)¶

Bases: Labelframe

GUI component for loading an architecture (.arch) file and related resources.

This frame provides tools to: - Load an architecture definition file (.arch) - Load a compiler module (Python file) - Load a logical or open noise model - Optionally inspect the QPU state (if Bovary tools are enabled)

It serves as the top section of the Bovary interface and updates the main application status bar upon successful loading of files.

load_compiler()¶

Open a file dialog to select and load a compiler module.

The selected Python source file is passed to SubGUI.load_compiler_from_path(), which dynamically imports and integrates the compiler with the loaded architecture.

load_file()¶

Open a file dialog to load an architecture definition (.arch).

This function delegates to SubGUI.load_architecture() and updates the displayed architecture name after successful loading.

class Bovary.bovary.CircuitPreviewFrame(parent)¶

Bases: Labelframe

Visual frame displaying the quantum circuit layout.

This component provides an interactive visualization of the currently loaded quantum circuit, including all gates, controls, and target qubits. It supports: - Scrollable and zoomable canvas for circuit exploration. - Context menu for QPU state inspection tools. - A timeline viewer for gate-by-gate QPU evolution.

clear()¶

Clear all elements from the circuit canvas.

Resets the canvas and all internal tracking structures, preparing for a complete redraw (e.g., when reloading a new circuit).

insert_gate(gate_dict, op_index)¶

Draw a single gate element on the circuit canvas.

This method renders a gate at the next available column position based on its type (Logical, Tape, Analog, Error, or Separator). Control and target qubits are visually connected as needed.

Parameters:

  • gate_dict (dict) – Dictionary containing gate data with keys gate, controls, targets, theta, etc.

  • op_index (int) – The index of the gate in the circuit (used for event binding).

Notes

  • Separator gates create vertical dashed lines across all rails.

  • Analog and Tape gates span multiple qubits vertically.

  • Double-clicking a gate opens the QPU Inspector for that step.

prompt_param_overrides_for_gate(gate_name)¶

Prompt the user for analog parameter overrides for a given gate.

This method looks up all parametric pulse constants for the specified gate and, if applicable, opens a dialog window allowing the user to edit them.

Parameters:

gate_name (str) – The name of the gate for which parameter overrides are requested.

Returns:

A dictionary of overridden parameter values provided by the user. Returns an empty dictionary if the gate has no parametric constants.

Return type:

dict

redraw_circuit(circuit, hide_tape=False, hide_error=False)¶

Redraw the entire quantum circuit on the canvas.

Clears all existing drawings and rebuilds the visual layout for every gate and qubit rail according to the circuit definition.

Parameters:

  • circuit (list[dict]) – List of gate dictionaries containing gate, type, controls, targets, etc.

  • hide_tape (bool, optional) – If True, hides Tape-type gates from the visualization.

  • hide_error (bool, optional) – If True, hides Error-type gates from the visualization.

set_error_gates_grayed(gray=True)¶

Gray out Error gate rectangles while keeping text labels visible.

set_metadata(metadata, num_qubits)¶

Assign the circuit metadata and initialize the base qubit lines.

Parameters:

  • metadata (dict) – Dictionary mapping gate names to their associated metadata, such as symbols, types, and layout information.

  • num_qubits (int) – The total number of qubits to be displayed.

class Bovary.bovary.GateSelectorFrame(parent)¶

Bases: Labelframe

Side panel for selecting and managing quantum gates.

This frame allows the user to: - Select a gate category (Logical, Tape, Analog, Error) - Choose control and target qubits - Specify optional parameters (e.g., rotation angle θ) - Add or remove gates in the circuit - Insert visual separators for readability

The entire frame content is scrollable vertically to support long lists of gates.

add_gate()¶

Add a selected gate to the circuit.

This method collects the user’s selections (gate type, control qubits, target qubits, and optional parameters), validates them, and then inserts the gate into the circuit. It handles both compiler-based and manual gate addition logic.

Notes

  • Validation ensures the number of control/target qubits respects the gate’s allowed configuration.

  • For Error gates, manual insertion is disabled when general noise mode is active.

  • If a compiler is loaded, its get_gate_list() method is called to expand the input into one or more lower-level gates.

  • Analog gates can trigger a parameter override dialog.

add_separator()¶

Insert a visual separator (barrier) into the circuit.

This method adds a purely visual divider between groups of gates to improve circuit readability. It does not affect the QPU state or simulation timeline.

Notes

  • Delegates to the parent SubGUI controller.

  • Used for organizational clarity in both the circuit preview and QASM export.

cancel_gate()¶

Remove the selected gate from the circuit list.

This method deletes the currently highlighted gate in the sidebar and updates all relevant views in the main application.

on_gate_selected(event=None)¶

Show or hide the rotation angle (θ) input field depending on the gate type.

For parametric rotation gates such as RX, RY, or RZ, this field becomes visible so the user can enter a custom angle value.

populate_gate_list(gates)¶

Populate the gate dropdown with a provided list of gates.

Parameters:

gates (list[str]) – The list of gate names to display in the combobox.

populate_qubit_list(max_qubits)¶

Populate the control and target qubit listboxes.

Parameters:

max_qubits (int) – The maximum number of qubits available in the architecture.

set_controller(controller)¶

Link the parent controller (SubGUI) to this frame.

Parameters:

controller (SubGUI) – The SubGUI controller that manages circuit operations.

set_gate_lists(logical_gates, tape_gates, analog_gates, error_gates=None)¶

Register the available gate lists for each category.

Parameters:

  • logical_gates (list[str]) – Logical gate names.

  • tape_gates (list[str]) – Tape gate names.

  • analog_gates (list[str]) – Analog gate names.

  • error_gates (list[str], optional) – Error gate names (default: empty list).

update_gate_list()¶

Refresh the gate selection dropdown based on the chosen category.

This method repopulates the combobox with the gates corresponding to the selected radio button (Logical, Tape, Analog, or Error).

Notes

  • Automatically falls back to Logical gates if no valid category is selected.

  • Maintains backward compatibility with architectures that may not define every category.

class Bovary.bovary.SubGUI(master: Widget, arch_file: str | None = None, compiler_path: str | None = None, noise_model_type: str | None = None, noise_model_path: str | None = None)¶

Bases: Frame

Embedded Bovary sub-GUI (to be inserted in the parent ArchBuilder window).

This frame provides: - Architecture loading and status - Gate browsing by category and circuit construction - Scrollable circuit preview with separators and measurement support - QASM code view/export - Optional compiler integration for validation and composite expansion - Optional noise model editing/loading (logical QEC) and open-system mode - Utilities to visualize layouts and pulses

Notes

This widget does not create a Tk root on its own; it must be embedded in an existing application window.

The class intentionally keeps heavy initialization in methods so that AutoDoc can document each piece and so unit tests can exercise them independently.

add_gate_to_circuit(gate, theta, controls, targets, param_overrides=None)¶

Add a new gate to the circuit and refresh all dependent views.

This method appends a gate operation (and its metadata) to the internal circuit list, updates the circuit preview visualization, recomputes the QPU timeline, and regenerates the corresponding QASM code.

Parameters:

  • gate (str) – Name of the gate to add.

  • theta (float or None) – Optional gate rotation angle (for parametric gates).

  • controls (list[int]) – Indices of control qubits (if any).

  • targets (list[int]) – Indices of target qubits for the operation.

  • param_overrides (dict, optional) – Optional parameter overrides for analog gates.

add_separator()¶

Insert a visual barrier (Separator) into the circuit.

This creates a non-executable marker within the circuit used purely for readability and organization — equivalent to a barrier statement in QASM.

Notes

  • It does not affect QPU simulation or recomputation.

  • The separator appears in both the preview and the textual QASM view.

build_qasm_from_circuit()¶

Generate QASM source code from the current circuit.

This method reconstructs an OpenQASM 2.0 program from the currently loaded circuit, automatically adding architecture metadata and compiler information as structured header comments.

Features¶

  • Includes architecture, layout, and compiler metadata in the header.

  • Expands compiler-defined composite gates (macros) into their primitive gate sequences.

  • Converts Separator gates into QASM barrier statements.

  • Automatically generates meas statements for measurement gates.

  • Supports broadcasting of Tape or Analog gates across all qubits when no specific targets are provided.

returns:

A string containing the complete QASM program, ready to be saved or displayed in the code editor.

rtype:

str

edit_noise_model()¶

Open the QEC Logical Noise Model Editor in a new window.

If a noise model is already loaded, it is pre-populated in the editor. When the editor supports an on_export callback (newer versions), the model is reloaded automatically once saved.

If the editor is an older version that lacks the callback mechanism, the user is prompted to import the exported JSON file manually after closing the editor window.

export_pulses()¶

Export the current pulse sequence to an .operseq file.

This creates a simplified list of the pulse operation identifiers (gates) currently in the circuit and saves them as a JSON file.

Notes

  • Separators (__SEP__) are skipped.

  • The exported format contains a single key "oper_list".

  • The file can later be used by Metamorphosis or external analysis tools.

hide_error()¶

Toggle visibility of Error gates in the circuit preview.

This method switches between showing and hiding all Error-type gates in the CircuitPreviewFrame. It updates the circuit display immediately without modifying the underlying circuit data.

hide_tape()¶

Toggle visibility of Tape-type gates in the circuit preview.

This toggles the display of Tape gates on and off in the CircuitPreviewFrame without altering the circuit data itself.

load_architecture(filepath)¶

Load an architecture (.arch) file and initialize internal data structures.

This method loads a JSON architecture definition, parses its associated layout and operation files, and populates all relevant metadata used by the Bovary interface. It also constructs categorized lists of gates (Logical, Tape, Analog, Error) based on the discovered .oper files.

Parameters:

filepath (str) – Path to the .arch file to load.

Returns:

The name of the loaded architecture (family_version format).

Return type:

str

Notes

  • The method automatically looks for a corresponding layout JSON file referenced inside the .arch metadata.

  • All gate metadata (.oper files) are scanned dynamically and registered.

  • If the architecture does not define certain fields (e.g. layout_path), defaults are applied.

load_compiler_from_path(path)¶

Load a compiler module dynamically from a given Python file path.

This method imports a compiler module (typically a Python file defining QPU-specific compilation rules) and integrates it into the Bovary environment. It automatically registers any composite operations defined by the compiler and updates all gate lists accordingly.

Parameters:

path (str) – Path to the Python source file containing the compiler module.

Returns:

True if the compiler was successfully loaded, False otherwise.

Return type:

bool

Notes

  • The compiler may define composite operations in COMPOSITES that will appear as virtual gates in the GUI.

  • If available, the compiler’s register_composites_with_arch() function is called to synchronize composite definitions with the architecture.

load_noise_model_dialog()¶

Open a file selection dialog to load a noise model JSON file.

This method opens a standard file dialog that allows the user to select a noise model file (.json). Once selected, it loads the model into Bovary and registers all defined gates (logical or error gates) accordingly.

load_noise_model_from_path(path: str) → bool¶

Load a QEC noise model JSON file and register its error gates in Bovary.

This function supports both: - Logical noise models — symbolic error models defined via logical gates. - Open-system noise models — Lindblad-style dissipative models.

The method automatically updates the user interface according to the model type and enables the noise activation toggle when loading is successful.

Parameters:

path (str) – Path to the noise model JSON file.

Returns:

True if the model was successfully loaded, False otherwise.

Return type:

bool

load_qasm()¶

Load a QASM file and reconstruct the corresponding circuit.

This method reads a QASM file (.qasm) exported by Bovary or another compatible tool and rebuilds the internal circuit structure. It validates the target architecture, reconstructs gates, and updates all visual and textual circuit representations.

Notes

  • A QASM header line // Target: ARCH_NAME must be present.

  • Architecture names must match, otherwise loading is aborted.

  • barrier commands are converted into visual separators.

  • Unrecognized gates are skipped gracefully.

on_close()¶

Handle the event when the Bovary window is closed.

This method ensures that, if a noise model is currently loaded, its state and file path are communicated back to the parent ArchBuilder application. This allows ArchBuilder to automatically update the .arch project metadata with the latest noise model information before Bovary is destroyed.

open_noise_model_editor()¶

Open the QEC logical noise model editor.

If the editor supports an ‘on_export’ callback, Bovary will auto-load the resulting JSON and expose its Error gates. Otherwise, after the editor closes, the user is prompted to pick the saved JSON.

open_qpu_inspector()¶

Open a full-circuit QPU state inspector.

This launches the QPU Inspector window showing a full snapshot of the system state after executing all operations currently in the circuit.

Notes

  • The inspector is interactive: users can expand and collapse nested dictionaries and export the state to JSON.

  • The state can be refreshed dynamically using the built-in Refresh button.

open_qpu_inspector_at(index)¶

Open the QPU Inspector for the state after a specific operation index.

This method is typically triggered by double-clicking a gate or barrier in the circuit preview. It displays the QPU state after executing all operations up to the given index (0-based).

Parameters:

index (int) – The index (0-based) of the circuit operation after which to display the resulting QPU state.

Notes

  • The method automatically recomputes the QPU timeline if out of sync.

  • It gracefully handles missing compiler information or invalid indices.

plot_pulses()¶

Plot all pulse envelopes used in the current circuit.

This method generates a Matplotlib figure with subplots corresponding to each qubit species and drive channel (x, y, z). It supports multiple pulse definitions including: - Gaussian pulses (with variance) - Custom waveform files (NumPy .npy arrays) - Constant pulses (square segments)

Notes

  • Pulses are color-coded by species.

  • The plot shows time evolution for each channel (Ωx, Ωy, Ωz).

  • Vertical dashed lines indicate pulse start and end times.

prompt_param_overrides_for_gate(gate_name)¶

Prompt the user for analog parameter overrides for a specific gate.

This delegates to the CircuitPreviewFrame, which provides the actual user interface for modifying per-gate analog constants.

Parameters:

gate_name (str) – Name of the gate for which parameter overrides are requested.

Returns:

A dictionary of user-defined parameter overrides.

Return type:

dict

remove_gate_at_index(index)¶

Remove a gate from the circuit at a specified index.

This method deletes a gate entry safely and fully refreshes all dependent components, including: - Gate list sidebar - Circuit preview - QPU state timeline - Generated QASM code

Parameters:

index (int) – Index of the gate to remove (0-based).

Notes

  • If the index is out of bounds, the operation is silently ignored.

  • When a compiler is present, the QPU state is recomputed accordingly.

run_comparison_zhivago()¶

Run a pulse-sequence comparison between two layouts.

This performs a pairwise comparison using the same pulse sequence on two different architecture layout JSON files. The comparison computes the overlap between the two resulting states.

Notes

  • The first layout is automatically taken from the loaded architecture.

  • The user is prompted to select a second layout JSON file.

  • Both simulations reuse the same pulse sequence defined by the circuit.

  • Results include overlap magnitude and optional final-state previews.

run_simulation_zhivago()¶

Run a full pulse-sequence simulation using Metamorphosis/Žhivago.

This routine executes the list of pulse operations (.oper files) in the current circuit using the Julia backend (Metamorphosis module).

Notes

  • The simulation is performed on the layout associated with the loaded architecture.

  • Any temporary .oper files with modified pulse parameters are generated and cleaned up after execution.

  • Results are reported as a compact summary once complete.

save_qasm()¶

Save the currently generated QASM code to a file.

The content of the CodeEditorFrame text area is written to a .qasm file. The user is prompted for the destination path.

Notes

  • If no QASM code is available, a warning is displayed.

  • The saved file can be reloaded later for verification or reuse.

show_interaction_info(cid, parent)¶

Display information about a selected interaction.

When the user clicks an interaction line in the layout canvas, this method retrieves and displays the corresponding Hamiltonian term connecting the two degrees of freedom (DOFs).

Parameters:

  • cid (str) – Interaction key in the format "(i,j)" identifying the DOFs.

  • parent (tkinter.Toplevel) – Parent window in which to display the message box.

show_qubit_info(qid, parent)¶

Display information about a selected qubit.

When the user clicks on a qubit in the layout canvas, this method shows its physical and local Hamiltonian properties.

Parameters:

  • qid (int) – The identifier of the selected qubit.

  • parent (tkinter.Toplevel) – Parent window in which to display the message box.

toggle_noise_active()¶

Toggle activation of the currently loaded noise model.

This method enables or disables the active noise model depending on its type:

  • Logical mode: Shows or hides symbolic error gates in the GUI and updates the noise label color (gray when inactive, blue when active).

  • Open mode: Toggles the open-system (Lindblad) simulation flag.

  • General error mode: Displays a note when global stochastic noise is active, meaning error gates cannot be added manually.

Notes

If no noise model has been loaded, the function will warn the user and disable the toggle.

update_message_text()¶

Update the status message displayed in the architecture frame.

This method provides live feedback on the loading status of the architecture and compiler, updating the message color and content accordingly.

Status Codes¶

  • ✅ Green: Architecture and compiler are both loaded.

  • ⚠️ Orange: Only one of the two is loaded (partial configuration).

  • ❌ Gray: Neither compiler nor architecture is loaded.

view_layout()¶

Display the architectural layout of the QPU on a new canvas window.

This method extracts the layout information from the loaded architecture (or from available .oper files if not yet populated) and visualizes it in a Tkinter window. Each degree of freedom (DOF) — representing a qubit or resonator — is drawn as a colored circle, while factor nodes representing interactions are drawn as connecting lines.

Notes

  • The layout includes both local Hamiltonian terms and interactions.

  • If no explicit layout is found in the architecture, it attempts to infer one from any available .oper files.

  • The viewer supports zooming and panning with inertia for easy navigation.

Bovary.noise_model_editor module¶

QEC Logical Noise Model Editor¶

Graphical sub-GUI for defining logical error models at the quantum error-correction (QEC) level. Users can define theoretical error channels such as bit-flip (X), phase-flip (Z), or depolarizing errors, assign probabilities, and link them conceptually to operators—not physical .oper files.

The exported JSON describes a purely theoretical noise model, e.g.:

{
  "errors": {
    "bitflip": {
      "symbol": "Xerr",
      "operator": "X",
      "probability": 0.01,
      "description": "Logical bit-flip error"
    },
    "phaseflip": {
      "symbol": "Zerr",
      "operator": "Z",
      "probability": 0.02
    }
  }
}

ArchBuilder module¶

ArchBuilder main application interface.

This module defines the ArchBuilderApp, the central GUI frame of the ArchBuilder software. It provides integration between multiple Planckian modules (Livingston, Frankenstein, Bovary, Metamorphosis) and manages the plugin system, layout metadata, and project coordination logic.

Modules loaded:

  • Tkinter UI components

  • Pillow for image handling

  • JSON, ZIP, and temporary file utilities

  • Planckian plugin interfaces and submodules

build_interface()¶

Build and initialize the main user interface.

This method constructs all major UI components of the ArchBuilder application, including:

  • The top toolbar with project controls and plugin selection

  • The grid-based module launcher (Steppenwolf, Livingston, etc.)

  • The log output area and control buttons

The function connects interface elements to their respective callback functions and configures their initial states.

Notes

The layout grid follows the Planckian module pipeline: Steppenwolf → Livingston → Frankenstein → Bovary → Metamorphosis

Each module can be enabled or disabled dynamically depending on project state.

build_pulses_index_from_collected()¶

Build or refresh the pulses index file.

Parses all collected .oper files and generates a temporary pulses_index.json file listing valid operations and their associated species. Invalid or mismatched species are logged.

Notes

The generated index file is written inside the temporary project directory (self.temp_dir).

plugin_manager module¶

Plugin management utilities for Planckian ArchBuilder.

This module defines the core classes for discovering and executing modular plugins. Plugins are expected to reside in subdirectories under /modules and contain a plugin.py file with:

  • a global metadata dictionary defining name, version, author, etc.

  • a register(app) function used for integration.

  • arch_path (str, optional) – The path of the saved or modified .arch file.

  • metadata (dict, optional) – Updated architecture metadata dictionary.

  • saved (bool, optional) – If True, the project has just been saved and should be marked clean. If False, the project has unsaved modifications.

on_bovary_closed(noise_model_path=None, noise_active=False, model_dict=None)¶

Callback invoked when Bovary closes.

Updates .arch metadata with the logical or open noise model path and marks the architecture as modified (unsaved) in the UI.

on_oper_selected(event=None)¶

Event handler for operation dropdown selection.

Parameters:

event (tkinter.Event, optional) – The triggering event (not used).

on_steppenwolf_export(data)¶

Handle incoming data from the Steppenwolf module.

Updates internal project structures with the received circuit, Hamiltonian, coefficients, and other metadata.

Parameters:

data (dict) – Dictionary containing Steppenwolf export data such as circuit paths, Hamiltonians, and coefficients.

open_arch_creator()¶

Open the ArchCreator window for editing the .arch configuration file.

This function launches the ArchCreatorWindow, allowing the user to modify or inspect architecture metadata. If the current metadata is incomplete, it attempts to reconstruct it using the existing layout file.

Notes

If species or layout information is missing from the current arch_metadata, this method will attempt to extract it directly from the loaded layout JSON file.

open_frankenstein(layout_dict=None, oper_path=None)¶

Open the Frankenstein pulse editor window.

Launches a new top-level Tk window containing a PulseEditorApp instance for editing and visualizing pulse operations associated with the current architecture.

Parameters:

  • layout_dict (dict, optional) – Layout data to pass to the pulse editor (default is None).

  • oper_path (str, optional) – Path to the .oper file being edited (default is None).

open_in_bovary()¶

Launch the Planckian Bovary Pulse Composer.

Opens a new window running the Bovary.SubGUI interface, preloading the current .arch file, compiler, and noise model if available. Handles both Logical (JSON-based) and Open (layout-based) noise models.

open_project()¶

Open an existing .arch project folder.

Prompts the user to select a project directory and loads the contained files and metadata into the interface. Updates the window title and unsaved-state indicator.

rebuild_pulse_index()¶

Rebuild the pulse index from the collected operations list.

This function revalidates .oper files based on the current architecture species and writes a fresh pulses_index.json file.

refresh_arch_metadata_from_layout(layout_path)¶

Synchronize architecture metadata with the selected layout file.

This method extracts layout information such as species and family, updating the internal arch_metadata dictionary accordingly.

Parameters:

layout_path (str) – Path to the layout JSON file.

refresh_oper_dropdown()¶

Refresh the operator dropdown list with valid operation files.

Populates the dropdown with all existing .oper files found in the collected operations list, updating both paths and selection.

reset_project()¶

Reset the current project to its initial state.

This method clears all loaded files, metadata, and visual links, resetting the ArchBuilder interface. It also disables relevant buttons and resets internal project variables. The window title and unsaved-state indicator are also cleared.

run_selected_plugin()¶

Execute the currently selected plugin.

The plugin is selected from the dropdown list and executed using the PluginManager.

run_simulation()¶

Launch the Planckian Metamorphosis simulator.

Opens a simulation window running the MetamorphosisApp, configured with the currently loaded project files.

Notes

  • Requires at least a valid layout file.

  • Optional files (pulses and dynamics) are used if available.

  • Simulation results are stored under self.project_files[“simulation_results”].

save_as_arch()¶

Export the current project as a .arch archive.

Packages all files in the current project folder into a zip archive with the .arch extension.

Notes

The .arch format is a compressed project structure that can be reopened directly by ArchBuilder.

save_log()¶

Save the contents of the log window to a text file.

Opens a save dialog, allowing the user to export the log output to a .txt file for later reference. If the log is empty, a notification is shown instead.

save_project()¶

Save the current project to its designated folder.

Copies all relevant JSON data files (layout, pulses, dynamics, etc.) from their current locations into the project directory, and writes the collected operations list as collected_operations.json.

Notes

  • The project must have a valid folder path (self.project_path).

  • Layout, pulses, and dynamics files are required to proceed.

scan_oper_files(folder)¶

Scan a folder for .oper pulse operation files.

Parameters:

folder (str) – Path to the folder to scan.

Notes

Creates or updates the pulses_index.json file from valid .oper entries and updates the dropdown for quick selection.

show_plugin_info()¶

Display metadata and information about the selected plugin.

Opens a popup window containing plugin details such as name, author, version, description, and target modules.

toggle_link(name, button)¶

Toggle a logical or visual link between modules.

Parameters:

  • name (str) – The connection identifier.

  • button (tk.Button) – The arrow button widget controlling this link.

Notes

Ensures mutual exclusivity between certain connections (e.g., LV↓ME and BO→ME) to avoid conflicting states.

update_buttons()¶

Update interface buttons and status indicators.

Refreshes the enabled/disabled states of key controls, visual indicators for loaded modules, and checklist labels.

class AIgor.aigor.UnitaryEditorApp(root: Tk)¶

Bases: object

Graphical interface for building and managing quantum unitaries.

This class implements the main AIgor window for composing unitary operations based on analytic or file-based definitions of quantum pulses. It integrates visualization, file management, and interactive operator assembly into a unified Tkinter interface.

build_H0()¶

Construct the static (local) Hamiltonian H₀.

Notes

This method builds:

\[H₀ = \sum_i \sum_k (\text{strength}_k) \; O_k^{(i)}\]

using each qubit’s local Hamiltonian terms stored in self.qubits[q]['local_Ham'] as a list of tuples: [(op_names, strength), ...].

Returns:

The complete static Hamiltonian in CSR sparse format.

Return type:

qutip.Qobj

Example

For a single qubit with local_Ham = [(["Z"], 1.0), (["X"], 0.2)], this returns:

\[H₀ = 1.0 * σᶻ + 0.2 * σˣ\]
build_Hdrive(allowed_pulses: dict) → list¶

Construct the drive Hamiltonians for each allowed species and operator.

Parameters:

allowed_pulses (dict) –

Dictionary specifying which pulse operators are enabled for each species. Example:

allowed_pulses = {
    "LQ": ["X", "Y"],
    "Cavity": ["a", "a†"]
}

Returns:

List of drive Hamiltonian terms (in CSR sparse format) for each active operator type across all species.

Return type:

list[qutip.Qobj]

Notes

  • Each species may include a subset of {X, Y, Z, a, a†, a+a†}.

  • The resulting operators are pre-scaled by their respective Rabi factors.

  • For qubits, rabi defaults to 10 if not defined in self.qubits[q].

  • The Hilbert space composition follows the system ordering defined in self.qubits.

Example

For one qubit species LQ with ['X', 'Y'] allowed, the result will include [Hₓ, Hᵧ] where:

Hₓ = ∑_q rabi * RabiFactor_q * σₓ^(q)
Hᵧ = ∑_q rabi * RabiFactor_q * σᵧ^(q)
build_Hint()¶

Construct the interaction Hamiltonian H_int.

Notes

This method builds:

\[H_{\text{int}} = \sum_{i<j} \sum_α (\text{strength}_α) (O_i ⊗ O_j ⊗ …)\]

using entries in self.interactions. Each entry is defined as:

self.interactions["(0,1)"] = [
    ["X0 ⊗ Y1", 0.5],
    ["R(θ=(1/2)*pi,φ=0.25)0 ⊗ Z1", 0.1]
]
Returns:

Interaction Hamiltonian in CSR sparse format.

Return type:

qutip.Qobj

Notes

  • Each operator string (op_str) can include multi-qubit tensor expressions, e.g. "X0 ⊗ Y1".

  • Supports standard Pauli, projectors, creation/annihilation, and rotation operators R(θ, φ).

  • Unrecognized operator symbols raise NotImplementedError.

build_unitary()¶

Construct the total system unitary from stored component operations.

qutip.Qobj

Full composite unitary acting on the complete Hilbert space, in CSR (compressed sparse row) format.

  • Qubits are ordered by their integer IDs.

  • Each local operator dictionary in self.unitaries defines a set of terms with optional coefficients.

  • The final operator is built as a product of summed tensor terms:

    :math:`U_{total} = prod_i left( sum_j c_j O_j

ight)`

  • Example of a unitary term:

    {"coeff": "1.0", "0": "X", "1": "Z"}

    which corresponds to 1.0 * (X ⊗ Z).

delete_selected_segment()¶

Delete selected unitary entries from the table and internal storage.

Notes

  • Removes entries from both the Treeview and self.unitaries.

  • Automatically refreshes the display afterward.

find_optimal_pulses()¶

Open the Find Optimal Pulse dialog window.

Notes

  • Provides a graphical interface for configuring pulse optimization runs.

  • Allows setting simulation time, saving path, and temporal discretization mode.

  • Supports both adaptive and fixed time-stepping strategies.

  • The second part of this method (defined later) performs the optimization itself.

Raises:

Warning – If no layout has been loaded before launching the optimization.

load_layout(layout=None, file_path=None)¶

Load a Livingston layout JSON file and extract qubit and interaction data.

Parameters:

  • layout (dict, optional) – A preloaded layout dictionary. If provided, file selection is skipped.

  • file_path (str, optional) – Path to a layout file (.json). If layout is None and no path is given, opens a file dialog for selection.

Notes

  • Initializes the following attributes: * self.qubits * self.interactions * self.species * self.H0 (local Hamiltonian) * self.Hint (interaction Hamiltonian)

  • Each DOF entry is converted into an internal qubit record with position, color, species, Hilbert dimension, and local Hamiltonian terms.

load_unitary_file()¶

Load a previously saved .unit file and restore the layout and unitaries.

Notes

  • Rebuilds the layout and reloads all unitary definitions into the table.

  • Automatically updates the next available unitary ID counter.

load_unitary_table(event=None)¶

Populate the Treeview table with the currently defined unitary tensors.

Parameters:

event (tkinter.Event, optional) – Reserved for event bindings (unused here).

Notes

  • Each unitary is represented as a sum of tensor products.

  • The method reconstructs the full tensor string representation from self.unitaries.

  • The row height of the Treeview dynamically adjusts to fit multiline tensor definitions.

Example

If self.unitaries contains:

{
    "U1": [
        {"coeff": "1.0", 0: "X", 1: "I"},
        {"coeff": "0.5j", 0: "Y", 1: "Z"}
    ]
}

The table will show:

(1.0)*X_q0 ⊗ I_q1
+ (0.5j)*Y_q0 ⊗ Z_q1
open_tensor_builder()¶

Open the Tensor Builder window for creating composite unitaries.

Notes

  • This GUI lets the user define unitary terms as tensor products of single-qubit or multi-mode operators.

  • Each operator can be selected individually or by species (batch mode).

  • Supports standard operators (I, X, Y, Z, H, Pₑ, P_g), rotations R(θ, φ), and custom bosonic operators like a, a†.

Raises:

Warning – If no layout has been loaded prior to launching the builder.

save_unitary_file()¶

Save the current operation and layout as a .unit file.

Notes

  • Requires a valid operation name and loaded layout.

  • Saves both the layout and all defined unitaries in JSON format.

  • The .unit file can later be reloaded using load_unitary_file().

File Structure¶

{
    "metadata": {"version": "v2", "tool": "Planckian AIgor"},
    "operation": "MyOperation",
    "unitaries": {...},
    "layout": {...}
}

AIgor.aigor.build_tensor_operator(term_dict: dict, qubit_ids: list[int], dims: list[int])¶

Construct the full tensor product operator from a term definition.

Parameters:

  • term_dict (dict) – Mapping from qubit ID (as str) to operator string, e.g. {"0": "X", "1": "Z"}.

  • qubit_ids (list[int]) – Ordered list of qubit IDs defining tensor order.

  • dims (list[int]) – Hilbert-space dimensions for each qubit in qubit_ids.

Returns:

Tensor product operator acting on the full system.

Return type:

qutip.Qobj

Notes

  • Missing qubit entries default to identity (I).

  • All operators are converted to CSR format for efficiency.

  • Example:

    term = {"0": "X", "1": "Z"}
op = build_tensor_operator(term, [0, 1], [2, 2])
AIgor.aigor.parse_angle(value_str: str, mode: str) → float | None¶

Parse a textual angle value expressed as a fraction, decimal, or π multiple.

Parameters:

  • value_str (str) – String representation of the angle, such as "1/2", "3*pi/4", or "0.75". Whitespace is ignored.

  • mode (str) –

    Parsing mode that determines whether to multiply the result by π. Accepted values:

    • "pi": multiply the parsed number by π

    • "number": interpret the value directly as radians

Returns:

The evaluated angle in radians, or None if parsing fails.

Return type:

float or None

Raises:

ValueError – If the string cannot be parsed as a valid number or fraction.

Notes

  • Supports fractional inputs (e.g., "1/4" → 0.25).

  • When mode == "pi", the function multiplies the value by NumPy’s π constant.

  • On invalid input, prints an error and returns None instead of raising.

  • This function is commonly used by AIgor to convert user-entered angles in GUI fields to numeric radians before building quantum rotation gates.

AIgor.aigor.parse_single_op(op_str: str, dim: int)¶

Parse a single operator string and return the corresponding QuTiP operator.

Parameters:

  • op_str (str) – String representing a quantum operator (e.g. “X”, “Y”, “Z”, “R(θ=pi/2,φ=pi/2)”).

  • dim (int) – Hilbert space dimension of the target system (usually 2 for qubits).

Returns:

The corresponding operator in CSR (compressed sparse row) format.

Return type:

qutip.Qobj

Raises:

  • ValueError – If the operator string cannot be recognized or parsed.

  • Supported Operators –

  • ------------------- –

:raises - "I" : Identity: :raises - "X", "Y", "Z" : Pauli matrices: :raises - "H" : Hadamard (normalized as (X + Z)/√2): :raises - "P_g", "P_e" : Projectors |g⟩⟨g| and |e⟩⟨e|: :raises - "R(θ, φ)" : Generic single-qubit rotation: Example: R(θ=pi/2, φ=pi/4) implements: \(R(θ, φ) = cos(θ/2) I - i sin(θ/2) (cos φ X + sin φ Y)\) :raises - "a", "a†" : Bosonic annihilation/creation operators: (products are parsed left to right).

Notes

  • All returned operators are cast to CSR sparse format using .to(‘CSR’).

  • Fractional π notation is supported in R(θ, φ) expressions.

  • Example:

    op = parse_single_op("R(θ=1/2*pi,φ=pi/2)", 2)
